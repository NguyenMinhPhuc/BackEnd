BUỔI 3: CẤU TRÚC DỰ ÁN EXPRESS CHUYÊN NGHIỆP

Từ Code lộn xộn đến Kiến trúc Phân lớp (MVC)

Giảng viên: Thầy Nguyễn Minh Phúc

Chuẩn đầu ra (LLOs)

  • Kiến thức: Giải thích được vai trò của từng thư mục trong mô hình MVC/Layered Architecture.
  • Kỹ năng: Điều khiển AI khởi tạo cấu trúc Boilerplate chuyên nghiệp và tách code thành các Module.
  • Thái độ: Hình thành thói quen tổ chức mã nguồn khoa học ngay từ khi bắt đầu.

Vấn đề của "Spaghetti Code"

  • Viết tất cả logic vào file server.js.
  • Khó bảo trì, khó đọc khi dự án có 100+ API.
  • Khó làm việc nhóm vì mọi người đều sửa chung 1 file.
  • Giải pháp: Chia để trị (Separation of Concerns).

Mô hình MVC là gì?

  • Model: Quản lý Dữ liệu (SQL Server).
  • View: Giao diện người dùng (Vite/React).
  • Controller: Bộ điều khiển (Cầu nối xử lý logic).
  • Học phần này tập trung vào M và C.

Kiến trúc 3 lớp (3-Tier)

  • Routes: Định nghĩa các con đường (Endpoint).
  • Controllers: Nhận Request & Trả Response.
  • Services: Xử lý Logic nghiệp vụ (Tính toán, DB).
  • Lợi ích: Dễ dàng thay thế SQL Server bằng DB khác mà không hỏng cả hệ thống.

Folder: `src/configs/`

  • Nơi chứa các cấu hình hệ thống: Database, Biến môi trường, Mail server.
  • Giúp tách biệt thông tin cấu hình khỏi mã nguồn logic.
// src/configs/db.js const mssql = require('mssql'); // Cấu hình từ .env

Folder: `src/routes/`

  • Định nghĩa danh sách các API URL.
  • Ví dụ: /api/users, /api/products.
  • Sử dụng express.Router() để module hóa.

Folder: `src/controllers/`

  • Nơi nhận dữ liệu từ Client (req.body, req.query).
  • Gọi đến Service để xử lý.
  • Trả kết quả về cho Client (res.json, res.status).

Folder: `src/models/`

  • Nơi định nghĩa cấu trúc bảng (Schema).
  • Quy định kiểu dữ liệu cho từng cột.
  • Buổi sau chúng ta sẽ dùng Sequelize tại đây.

Folder: `src/middlewares/`

  • Bộ lọc trung gian (Người gác cổng).
  • Kiểm tra xem User đã đăng nhập chưa?
  • Dữ liệu gửi lên có hợp lệ không?
  • Chạy trước khi vào Controller.

app.js vs server.js

  • app.js: Cấu hình Express, Middlewares, Routes.
  • server.js (hoặc index.js): Điểm bắt đầu (Entry point), nơi lắng nghe Port và kết nối DB.
  • Chia đôi để phục vụ việc Unit Testing sau này.

Review: Luồng đi của Request

  • Client $\rightarrow$ Server.js (Port) $\rightarrow$ app.js (Middlewares) $\rightarrow$ Routes $\rightarrow$ Controller $\rightarrow$ Service $\rightarrow$ Model.
