Tại sao phải biết JS Modern?

  • AI (Cursor/Copilot) luôn ưu tiên viết code theo cú pháp mới nhất (ES6+).
  • Code ngắn hơn nhưng khó đọc hơn nếu không có nền tảng.
  • Mục tiêu: Để "nói chuyện" và "sửa lưng" AI khi nó viết sai logic.

ES6+ Căn bản (AI hay dùng)

  • Arrow Functions: Cách viết hàm ngắn gọn.
  • Template Literals: Nối chuỗi bằng dấu backtick (`).
const hello = (name) => `Chào ${name}`; console.log(hello('Sinh viên LHU'));

Destructuring & Spread

  • Destructuring: Bóc tách dữ liệu từ Object/Array (Dùng cực nhiều khi nhận dữ liệu từ API).
  • Spread Operator (...): Sao chép hoặc gộp dữ liệu.
const user = { name: 'Phuc', role: 'Teacher' }; const { name } = user; // Bóc lấy name const newUser = { ...user, age: 30 }; // Gộp

Bất đồng bộ (Asynchronous)

  • Backend thường phải đợi: Đợi Database, đợi gửi Mail, đợi đọc File.
  • Nếu chạy đồng bộ (Sync): Web sẽ bị treo cho đến khi xong việc.
  • Giải pháp Node.js: Chạy việc khác trong lúc chờ (Non-blocking).

Async / Await

  • Cách viết code bất đồng bộ trông giống như đồng bộ.
  • Giúp code sạch, dễ đọc và dễ gỡ lỗi (debug).
async function getData() { const data = await database.fetch(); console.log(data); }

Review: Quản lý lỗi

  • AI rất hay bỏ qua `try...catch` nếu không yêu cầu.
  • Nguyên tắc: Mọi thao tác Database phải nằm trong khối xử lý lỗi.
Hỏi AI (Ctrl+L): "Đoạn code này sẽ thế nào nếu kết nối Database bị lỗi? Hãy thêm xử lý lỗi cho tôi."

Thiết kế Database (Cốt lõi)

  • Entities (Thực thể): Người dùng, Sản phẩm...
  • Relationships (Quan hệ): 1-1, 1-n, n-n.
  • Khóa (Keys): Primary Key (PK), Foreign Key (FK).

Chuẩn hóa dữ liệu (Normalization)

  • 1NF: Không có nhóm lặp (Dữ liệu nguyên tố).
  • 2NF: Phụ thuộc hoàn toàn vào khóa chính.
  • 3NF: Không phụ thuộc bắc cầu (VD: Đừng lưu Tên Tỉnh vào bảng Địa chỉ, hãy lưu ID Tỉnh).

Kiểu dữ liệu SQL Server

  • Chuỗi: NVARCHAR(max) cho tiếng Việt, VARCHAR cho mã/email.
  • Số: INT, DECIMAL(18,2) cho giá tiền.
  • Ngày: DATETIME2.
  • AI Tip: Yêu cầu AI "Sử dụng kiểu dữ liệu tối ưu cho SQL Server".

Mapping Dữ liệu (JS - SQL)

  • JS: `String` -> SQL: `NVARCHAR`
  • JS: `Number` -> SQL: `INT / DECIMAL`
  • JS: `Boolean` -> SQL: `BIT`
  • JS: `Object/Array` -> SQL: `JSON` (Hoặc tách bảng).
