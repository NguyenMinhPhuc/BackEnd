Backend Developer làm gì?

  • Quản lý Dữ liệu: Thao tác với SQL Server.
  • Bảo mật: Xác thực & Phân quyền.
  • Trung tâm điều khiển của mọi Web App.

Ví dụ thực tế: Khi bạn nhấn "Đặt hàng" trên Shopee, điều gì xảy ra?

  • Nhiệm vụ Backend
    • Xử lý logic (Tính tiền, giảm giá).
    • Giao tiếp với Database (Kiểm tra kho hàng).
    • Bảo mật (Xác thực người dùng).

Thông điệp: Backend là bộ não, Frontend là bộ mặt.

Mô hình Client - Server

Khái niệm

  • Client (Khách): Trình duyệt (Chrome, Safari), App Mobile. Gửi yêu cầu (Request).
  • Server (Chủ): Máy chủ chạy code Node.js. Trả kết quả (Response).

Ví dụ: Cách thức hoạt động

  • Client: Trình duyệt gửi Request.
  • Server: Node.js xử lý và gửi Response.
  • Khách hàng (Client) --> Người phục vụ (Request) --> Nhà bếp (Server) --> Món ăn (Response)

Mô hình Client - Server (Phần 2) - Giao thức HTTP

Ngôn ngữ chung: HTTP (HyperText Transfer Protocol).

Cấu trúc một request:

  • URL: Địa chỉ (ví dụ: /api/login).
  • Method: GET, POST, PUT, DELETE.
  • Status Codes: 200, 404, 500.
  • Body: Chứa dữ liệu (JSON).
  • Headers: Thông tin bổ sung (Kiểu dữ liệu, Token).
  • Body: Dữ liệu gửi đi (Username, Password).

Tại sao Node.js & Express?

    Node.js :
  • Môi trường chạy JavaScript trên máy tính (không cần trình duyệt).
  • Ưu điểm: Nhanh, nhẹ, dùng chung ngôn ngữ JS với Frontend.
    Express.js:
  • Thư viện (Framework) giúp xây dựng Server cực nhanh.
  • Vào đề: Nếu Node.js là động cơ, Express chính là khung xe hoàn chỉnh.

Cách mạng AI Lập trình

    Thay đổi tư duy:
  • Trước đây: Học thuộc cú pháp, gõ từng dòng lệnh.
  • Hiện nay: Học tư duy hệ thống, AI viết code - Lập trình viên kiểm duyệt.
    Công cụ chính:
  • GitHub Copilot: Trợ lý ảo gợi ý code ngay khi gõ. xác.
  • Cursor AI: Một IDE (trình soạn thảo) tích hợp AI cực mạnh, có thể đọc toàn bộ dự án để viết code chính
    Lợi ích:
  • Tăng năng suất gấp 5-10 lần.
  • Tập trung vào giải pháp hơn là cú pháp.

Giới thiệu Cursor AI

    Tại sao dùng Cursor?
  • IDE thế hệ mới dựa trên VS Code.
  • Tích hợp Claude 3.5 / GPT-4o.
  • Hiểu toàn bộ Project của bạn.
  • Khả năng @Symbols: Bạn có thể gọi AI đọc file Database để viết file API.
  • Tính năng Chat & Composer: Viết cả một tính năng chỉ bằng một yêu cầu.

Tư duy LHU: Chúng ta không dùng AI để lười biếng, chúng ta dùng AI để làm dự án phức tạp hơn trong thời gian ngắn hơn.

Kỹ năng "Nói chuyện với AI" (Prompt Engineering)

"Bạn là chuyên gia Node.js, hãy viết một API đăng ký..."
  • Công thức Prompt chuẩn cho Backend:

    Role + Context + Task.

    Trong đó:
  • Role (Vai trò): "Bạn là chuyên gia Node.js Senior..."
  • Context (Bối cảnh): "Tôi có Database SQL Server gồm các bảng..."
  • Task (Nhiệm vụ): "Hãy viết API đăng ký người dùng..."
  • Constraints (Ràng buộc): "Sử dụng Express, mật khẩu phải băm bằng Bcrypt..."
    Có thể bổ sung thêm:
  • "Think step by step before coding"
  • "First design database schema, then write code"
  • "Explain reasoning before implementation"
  • "Avoid pseudo code. Provide real production code."

Thực hành Setup môi trường (Demo)

    Các bước cài đặt:
  • Cài đặt Node.js (LTS Version).
  • Cài đặt Cursor AI. hoặc Github copilot chatbot.
  • Cài đặt SQL Server Management Studio (SSMS).

Bài tập thực hành tại lớp

  • Nhiệm vụ: Dùng Cursor AI để tạo một Server Express đơn giản.
  • Prompt gợi ý: "Hãy tạo một server Node.js sử dụng Express chạy ở cổng 5000. Có 1 route GET '/' trả về câu chào 'Welcome to LHU Backend Class'."
  • Yêu cầu: Sinh viên phải thay đổi nội dung câu chào và giải thích ý nghĩa của dòng lệnh app.listen().

Thực hành 1 - Cài đặt môi trường (The Base)

    Bước 1: Kiểm tra Node.js
  • Mở Terminal (cmd/powershell): node -v và npm -v..
  • Mục tiêu: Đảm bảo máy đã có môi trường chạy JavaScript.
    Bước 2: Cài đặt Cursor AI
  • Truy cập cursor.sh, tải và cài đặt.
  • Đăng nhập và chọn "Import Extensions" từ VS Code (nếu có).
    Bước 3: Cài đặt SQL Server (MSSQL)
  • Kiểm tra dịch vụ SQL Server đang chạy.
  • Mở SSMS để sẵn sàng kết nối.

Thực hành 2 - Khởi tạo dự án Node.js

    Thao tác trên Terminal của Cursor:
  • Tạo thư mục: mkdir lhu-backend-project
  • Di chuyển vào thư mục: cd lhu-backend-project
    mkdir backend-lhu
    cd backend-lhu
  • Khởi tạo dự án: npm init -y
  • Cài đặt Express: npm install express
Kiểm tra: File package.json xuất hiện trong thư mục.
Mẹo AI: Mở file package.json, bôi đen nội dung và nhấn Ctrl + L (Chat), hỏi AI: "Giải thích các thông số trong file này cho tôi".

Thực hành 3 - Viết Server đầu tiên với AI (The Prompt)

  • Bước 1: Tạo file server.js.
  • Bước 2: Sử dụng AI (Ctrl + K tại file server.js)
    Prompt: "Hãy viết một server Express cơ bản chạy ở cổng 5000. Tạo một route GET tại đường dẫn '/' trả về chuỗi JSON: { message: 'Chào mừng lớp liên thông LHU!' }. Log ra màn hình khi server chạy thành công."
  • Bước 3: Kiểm tra code AI vừa viết.
  • Bước 4: Chạy Server: * Gõ lệnh: node server.js
    • Mở trình duyệt truy cập: http://localhost:5000

Thực hành 4 - Cải tiến quy trình với Nodemon

  • Vấn đề: Mỗi lần sửa code lại phải tắt Server (Ctrl + C) và chạy lại node server.js. Rất mất thời gian!
  • Giải pháp: Cài đặt nodemon.
    Lệnh: npm install --save-dev nodemon
  • Cấu hình Scripts (Hỏi AI):
    Prompt: "Tôi vừa cài nodemon. Hãy hướng dẫn tôi cách thêm script 'dev' vào file package.json để tôi có thể chạy server bằng lệnh 'npm run dev'."
  • Thực hiện: Thay đổi nội dung file server.js và quan sát Server tự động khởi động lại.

Thực hành 5 - Review Code với AI (Quan trọng nhất)

  • Mục tiêu: Sinh viên không được copy-paste mà phải hiểu code.
  • Thao tác: Bôi đen đoạn code app.get(...) trong file server.js.
  • Nhấn Ctrl + L (Chat) và hỏi:
    • "Giải thích cho tôi tham số req và res đóng vai trò gì trong hàm này?"
    • "Tại sao phải dùng res.json() mà không dùng res.send()?"
  • Yêu cầu: Sinh viên ghi chú lại câu trả lời vào Nhật ký Prompt.

Thử thách tại lớp (Mini Challenge)

  • Yêu cầu: Tạo thêm một Route mới.
  • Nội dung:
    1. Đường dẫn: /api/info.
    2. Trả về thông tin của sinh viên gồm: { name: 'Tên của em', class: 'Lớp liên thông IT' }.
  • Cách làm: Tự viết code hoặc dùng AI gợi ý.
  • Kiểm chứng: Chụp màn hình kết quả chạy trên trình duyệt gửi vào nhóm lớp.
